5. Backend requirements in `backend/main.py`

The backend must:

5.1 Constants and API flags:

app = FastAPI(...)

OLLAMA_URL = "http://localhost:11434/api/generate"
LAYER_DIR = "../data/lens_rules"
TRANSLATION_DIR = "../data/translations"
QURAN_FILE = "../data/quran_arabic.json"

# To switch to Perplexity / OpenAI‑style APIs
USE_REMOTE_API = False   # set to True to use REMOTE_API_URL instead of Ollama
REMOTE_API_URL = "https://api.perplexity.ai/chat/completions"
REMOTE_MODEL = "llama‑2‑70b‑chat"
REMOTE_API_KEY = "pplx‑..."  # to be filled in by user

5.2 Endpoints:

- app.mount("/static", StaticFiles(directory="../frontend"), name="static")
- @app.get("/")
  Return FileResponse("../frontend/index.html")

- @app.post("/save_rules")
  Use Pydantic model SaveRulesRequest(layer: str, rules: str)
  - Save rules to: os.path.join(LAYER_DIR, f"{layer}.md")
  - Return: {"msg": "Rules saved", "path": path}

- @app.post("/run_bulk")
  Use Pydantic model RunBulkRequest(layer: str)
  - Read rules from LAYER_DIR/{layer}.md
  - Read QURAN_FILE as JSON
  - For each verse:
    - Build a prompt based on the rules text.
    - If USE_REMOTE_API:
      - Call REMOTE_API_URL (Perplexity‑compatible) with `system` + `user` prompt.
      - Extract assistant message content and append to results with:
        {"surah", "ayah", "arabic", "translated"}
    - Else:
      - Call OLLAMA_URL with `payload = {"model": "llama3.2:3b", "prompt": prompt, "stream": False}`
      - Use data.get("response", "...") and append to results as above.
    - On all errors, still log an error‑filled verse object so the JSON is valid.
  - Save full results to: os.path.join(TRANSLATION_DIR, f"{layer}.json")
  - Return:
    {
      "msg": "Bulk generation done",
      "output_path": "...",
      "results": results   # so the UI can consume it
    }

- (@app.get("/api/quran/surah/{surah_id}")
  Optional: pull surah verses from a free API such as:
  api_url = "https://alquran-api.pages.dev/api/quran/surah/{surah_id}?lang=ar"
  - Convert to [{"surah": ..., "ayah": ..., "arabic": ...}] and return.

5.3 Ensure all JSON is valid and safe for UI:
   - Never return `{"error": ""}` alone; every item in `results` must have at least:
     {"surah", "ayah", "arabic", "translated"} (or fallback text).
